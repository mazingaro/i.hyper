DESCRIPTION

The purpose of i.hyper.composite is to generate RGB, CIR, SWIR and custom composites from an existing hyperspectral raster_3d cube. Bands nearest to requested wavelengths are selected and auto-balanced with i.colors.enhance. Parameter input is an existing hyperspectral 3D raster map (voxel cube). Composites are generated by selecting the nearest bands to the requested wavelengths. Parameter output defines the base name for the output rasters (e.g., enmap → enmap_rgb, enmap_cir, …).

NOTES

RGB composites use i.colors.enhance with the -p flag to preserve relative colors. Other composites omit -p. Custom triplets can be provided as comma-separated wavelengths in nanometers. Temporary 2D rasters created from the cube are automatically removed, unless requested with -k.

EXAMPLE

i.hyper.composite input=enmap output=enmap composites=RGB,CIR,SWIR-agriculture
i.hyper.composite input=enmap output=enmap composites_custom=2200,848,572 strength=95
